## ğŸ’° Step 4: Implement Base Payment Backend

The Pricing page has a basic wallet connection, but you need to complete the payment flow.

### Option A: Manual Verification (Simplest)

1. User connects wallet
2. User sends USDC/ETH to your address
3. They email you the transaction hash
4. You verify on Basescan and manually activate their API key

**Your Base Wallet Address:** [You need to create one]

### Option B: Coinbase Commerce (Recommended)

1. Sign up at https://commerce.coinbase.com/
2. Create a charge for each pricing tier
3. Add webhook endpoint to activate API keys

```bash
npm install @coinbase/coinbase-commerce-node
```

Example backend code (Node.js):
```javascript
const Commerce = require('@coinbase/coinbase-commerce-node');
const Client = Commerce.Client;

Client.init('YOUR_COINBASE_COMMERCE_API_KEY');

// Create a charge
const chargeData = {
  name: 'Compute.io Starter Plan',
  description: '50,000 API requests/month',
  pricing_type: 'fixed_price',
  local_price: {
    amount: '29.00',
    currency: 'USD'
  }
};

const charge = await Charge.create(chargeData);
console.log(charge.hosted_url); // Send user here to pay
```

## ğŸŒ Step 5: Deploy to Production

### Deploy on Replit:

1. Click "Deploy" button in top right
2. Choose "Autoscale Deployment" (best for APIs)
3. Configure:
   - **Build Command:** `npm run build`
   - **Start Command:** `npm run preview` (or your server command)
   - **Port:** Usually 3000 or 5173

4. Wait for deployment (2-5 minutes)

### Set up custom domain:

1. Purchase domain (Namecheap, Google Domains, etc.)
2. In Replit deployment settings, add custom domain
3. Update DNS records at your registrar:
   ```
   Type: CNAME
   Name: @ (or www)
   Value: [Your Replit deployment URL]
   TTL: 3600
   ```
4. Wait for DNS propagation (5-60 minutes)

## ğŸ’³ Step 7: Payment Flow Implementation

### What needs to happen for payment:

1. **User clicks "Pay X USDC"**
2. **Wallet connects** (MetaMask/Coinbase Wallet)
3. **Switch to Base network** (if needed)
4. **Send transaction:**
   - For native ETH: Simple value transfer
   - For USDC: ERC-20 transfer to your address
5. **Wait for confirmation** (2-3 seconds on Base)
6. **Verify transaction** on your backend
7. **Generate and deliver API key**

### USDC Contract on Base:
```
Address: 0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913
```

### Example USDC Transfer Code:
```javascript
// USDC contract ABI (just the transfer function)
const usdcABI = [
  {
    "constant": false,
    "inputs": [
      { "name": "_to", "address": "address" },
      { "name": "_value", "uint256": "uint256" }
    ],
    "name": "transfer",
    "outputs": [{ "name": "", "type": "bool" }],
    "type": "function"
  }
];

// Send USDC
const web3 = new Web3(window.ethereum);
const usdcContract = new web3.eth.Contract(
  usdcABI, 
  '0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913'
);

await usdcContract.methods
  .transfer(
    'YOUR_WALLET_ADDRESS',
    web3.utils.toWei('29', 'mwei') // USDC has 6 decimals
  )
  .send({ from: userWalletAddress });
```

## ğŸ”‘ Step 8: API Key Generation

After payment is confirmed, you need to:

1. Generate a unique API key
2. Store it in your database with user info
3. Send it to the user via email
4. Display it in their dashboard

### Example API Key Generation:
```javascript
import crypto from 'crypto';

function generateApiKey() {
  const prefix = 'sk_live_';
  const random = crypto.randomBytes(32).toString('hex');
  return prefix + random;
}

// Example: sk_live_a1b2c3d4e5f6...
```
